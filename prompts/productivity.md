# Productivity Prompts — 20 Templates

## Automation & Workflows

### 1. Task Automation Identifier
```
Here's my typical work week: [DESCRIBE 5-10 RECURRING TASKS WITH TIME SPENT]. Tools I use: [LIST SOFTWARE/PLATFORMS]. Identify the top 5 tasks to automate ranked by (time saved × frequency). For each: (1) current manual process, (2) automation approach (Zapier, Python script, API, macro), (3) estimated setup time, (4) weekly time savings, (5) ROI in weeks. Provide a 30-day implementation roadmap.
```

### 2. Workflow Optimizer
```
Map and optimize this workflow: [DESCRIBE PROCESS — steps, people involved, handoffs, tools]. Current duration: [TIME]. Pain points: [LIST]. Redesign the workflow to: (1) eliminate unnecessary steps, (2) parallelize where possible, (3) reduce handoffs, (4) add automation triggers, (5) insert quality checkpoints. Show before/after flow diagrams (text format) with estimated time savings.
```

### 3. Email Rule & Filter System
```
I receive [NUMBER] emails/day. Categories: [LIST — newsletters, action items, FYIs, meetings, client communication]. I use [GMAIL/OUTLOOK/etc.]. Design a comprehensive email management system: (1) folder/label structure, (2) filter rules for auto-categorization (10 rules), (3) templates for common replies (5), (4) daily email processing workflow (touch-it-once method), (5) VIP list criteria, (6) unsubscribe criteria. Target: process all email in <30 min/day.
```

### 4. Personal Knowledge Management System
```
I learn about [TOPICS]. I consume [ARTICLES/BOOKS/VIDEOS/PODCASTS]. My tools: [NOTION/OBSIDIAN/DOCS/etc.]. Design a PKM system: (1) capture workflow — how to save interesting content in <10 seconds, (2) organization structure (folders, tags, links), (3) weekly review process, (4) spaced repetition for retention, (5) connection-making practices, (6) retrieval system for finding past notes. Keep it simple — max 15 min/day maintenance.
```

## Meeting Efficiency

### 5. Meeting Agenda Builder
```
I have a [DURATION]-minute [MEETING TYPE] with [NUMBER] attendees. Goal: [DECISION/BRAINSTORM/STATUS UPDATE/PROBLEM SOLVE]. Create: (1) pre-work assignments (what attendees should prepare), (2) timed agenda with facilitator notes, (3) parking lot setup, (4) decision-making framework for the meeting type, (5) action item template, (6) post-meeting summary template. Include a "do we even need this meeting?" checklist.
```

### 6. Meeting Notes to Actions Converter
```
Here are my meeting notes: [PASTE NOTES]. Extract: (1) decisions made (numbered, with rationale), (2) action items (who, what, by when), (3) open questions requiring follow-up, (4) key insights or quotes worth preserving, (5) parking lot items to schedule separately. Format as a structured summary email ready to send to attendees. Keep under 200 words.
```

### 7. Async Communication Drafter
```
I need to replace a [MEETING TYPE] with async communication. Information to share: [DESCRIBE]. Decisions needed: [LIST]. Deadline: [DATE]. Draft an async update using [SLACK/EMAIL/NOTION/LOOM] that: (1) provides context without a call, (2) clearly states what feedback is needed, (3) sets a decision deadline, (4) includes a response template so people can answer quickly, (5) has an escalation path if async doesn't resolve it.
```

## Project Management

### 8. Project Kickoff Document
```
Create a project kickoff document for [PROJECT NAME]. Scope: [DESCRIBE]. Team: [ROLES AND NAMES]. Duration: [TIMELINE]. Budget: [AMOUNT OR N/A]. Include: (1) project charter (objectives, success criteria, constraints), (2) RACI matrix, (3) milestone timeline (Gantt-style text table), (4) risk register (5 risks), (5) communication plan (who gets what updates, how often), (6) definition of done. One page maximum.
```

### 9. Sprint Planning Assistant
```
Plan a [DURATION]-week sprint. Team capacity: [STORY POINTS OR HOURS]. Backlog items: [LIST WITH ESTIMATES]. Dependencies: [DESCRIBE]. Sprint goal: [OBJECTIVE]. Generate: (1) selected items for sprint with justification, (2) capacity buffer (80% rule applied), (3) dependency sequencing, (4) risk items that might cause spillover, (5) daily standup focus areas for each day, (6) sprint retrospective questions tailored to last sprint's issues: [LIST].
```

### 10. Progress Report Generator
```
Generate a progress report for [PROJECT] — [WEEK/MONTH] [NUMBER]. Data: completed items = [LIST], in-progress = [LIST], blocked = [LIST], upcoming = [LIST]. Metrics: [VELOCITY/BURN-DOWN/BUDGET]. Audience: [STAKEHOLDERS/TEAM/EXECUTIVES]. Format: traffic light summary (overall RAG), key accomplishments (3), risks and mitigations (2), needs from leadership (1-2), next period focus. Keep under 300 words.
```

### 11. Retrospective Facilitator
```
Facilitate a retrospective for [TEAM/PROJECT]. Sprint/period: [NUMBER/DATES]. Known issues: [LIST WHAT CAME UP]. Team mood: [GOOD/MIXED/LOW]. Generate: (1) creative format (not just Start/Stop/Continue — suggest a themed approach), (2) warm-up question, (3) structured discussion prompts for each section, (4) voting/prioritization mechanism, (5) action item template with accountability, (6) follow-up check-in schedule. Duration: [MINUTES].
```

## Personal Productivity

### 12. Weekly Planning Template
```
Help me plan my week. Roles: [LIST — e.g., engineer, manager, parent, learner]. Top 3 priorities this week: [LIST]. Recurring commitments: [LIST WITH TIMES]. Available hours for deep work: [NUMBER]. Create: (1) weekly theme/intention, (2) time-blocked schedule with energy management (hard tasks when fresh), (3) daily top-3 priorities, (4) buffer blocks for unexpected work, (5) weekly review checklist for Friday.
```

### 13. Decision-Making Framework
```
Help me decide: [DESCRIBE DECISION]. Options: [LIST OPTIONS]. Constraints: [TIME/MONEY/RISK TOLERANCE]. Stakeholders affected: [LIST]. Apply a structured decision framework: (1) define success criteria (5 criteria with weights), (2) score each option against criteria, (3) identify reversibility of each option, (4) conduct pre-mortem — "if this fails in 6 months, why?", (5) recommendation with confidence level, (6) decision communication draft.
```

### 14. Learning Plan Builder
```
I want to learn [SKILL/TOPIC]. Current level: [BEGINNER/INTERMEDIATE/ADVANCED]. Available time: [HOURS/WEEK]. Deadline: [DATE OR OPEN-ENDED]. Learning style: [READING/VIDEO/HANDS-ON/MIXED]. Budget: [AMOUNT]. Create: (1) skill tree with dependencies, (2) curated resource list (free first, paid if significantly better), (3) weekly learning schedule, (4) project milestones to apply learning, (5) assessment checkpoints, (6) plateau-breaking strategies.
```

### 15. Habit Stack Designer
```
I want to build these habits: [LIST 3-5 HABITS]. Current routine: [DESCRIBE MORNING/EVENING]. Existing strong habits I never miss: [LIST]. Design a habit stack using the cue-routine-reward framework: (1) attach each new habit to an existing anchor habit, (2) start with 2-minute versions, (3) create a tracking system, (4) define the "never miss twice" recovery protocol, (5) 30/60/90 day progression plan, (6) environment design changes to reduce friction.
```

## Communication

### 16. Difficult Conversation Prep
```
I need to have a conversation about [TOPIC] with [RELATIONSHIP — manager, report, peer, client]. Situation: [DESCRIBE]. My goal: [DESIRED OUTCOME]. Their likely perspective: [WHAT THEY MIGHT THINK/FEEL]. Help me prepare: (1) opening statement using SBI framework (Situation-Behavior-Impact), (2) anticipated responses and my replies, (3) questions to ask to understand their view, (4) landing zone — what "good enough" looks like, (5) BATNA if conversation doesn't go well.
```

### 17. Presentation Outliner
```
Create a presentation outline for [TOPIC]. Audience: [DESCRIBE]. Duration: [MINUTES]. Goal: [INFORM/PERSUADE/INSPIRE]. Context: [FORMAL/INFORMAL]. Generate: (1) opening hook (3 options — story, statistic, question), (2) 3-5 main sections with key points, (3) transition sentences between sections, (4) data visualization suggestions per section, (5) closing with memorable takeaway, (6) Q&A preparation — 5 likely questions with draft answers. Slide count: [NUMBER].
```

### 18. Feedback Composer
```
Write feedback for [PERSON'S ROLE] about [TOPIC/BEHAVIOR]. Context: [PERFORMANCE REVIEW/1:1/REAL-TIME]. Tone: [DEVELOPMENTAL/APPRECIATIVE/CORRECTIVE]. Use the SBI-I model: (1) Situation — when and where, (2) Behavior — observable actions (not interpretations), (3) Impact — effect on team/project/outcome, (4) Intent — what you'd like to see instead. Keep under 100 words. Provide 2 versions: written and verbal talking points.
```

### 19. Delegation Brief
```
I'm delegating [TASK] to [PERSON/ROLE]. Their experience level: [JUNIOR/MID/SENIOR]. Task complexity: [LOW/MEDIUM/HIGH]. Deadline: [DATE]. Create a delegation brief: (1) task description with context (why it matters), (2) expected deliverable with quality criteria, (3) resources and access needed, (4) check-in schedule, (5) authority boundaries (what they can decide vs. escalate), (6) success criteria for the first milestone. Use the RACI model to clarify responsibilities.
```

### 20. End-of-Day Review Template
```
Generate a personalized end-of-day review template for a [ROLE]. Include: (1) wins — 3 things accomplished, (2) energy audit — what energized vs. drained, (3) tomorrow's top priority (just ONE), (4) someone to thank or acknowledge, (5) one thing learned, (6) inbox/Slack zero status check, (7) tomorrow's first action (decide tonight to avoid morning decision fatigue). Format as a 2-minute journal. Keep it under 10 fields — too many kills the habit.
```
